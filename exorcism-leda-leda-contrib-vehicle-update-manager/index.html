What this project does

Eclipse Leda is a complete Software Defined Vehicle reference stack — a Linux-based operating system distribution bundling all the Eclipse SDV components (KUKSA, Ankaios, Velocitas runtime, etc.) into a ready-to-flash image for automotive hardware like Raspberry Pi and QEMU.

Project: leda-leda-contrib-vehicle-update-manager  |  Build System: Go modules  |  Languages: Go

Why it exists in the SDV ecosystem

Setting up an SDV stack from scratch requires integrating dozens of components. Leda provides a validated, integrated distribution so teams can get started quickly.

Who uses it

Used by OEM prototype teams, automotive researchers, and developers building SDV proof-of-concepts.

6. Glossary

SDV Ecosystem Terms

TermDefinition
VSSVehicle Signal Specification — a hierarchical data model from COVESA that standardizes vehicle data signals (e.g., Vehicle.Speed, Vehicle.Cabin.Seat.Row1.Pos1.Position). Ensures all apps speak the same 'language' when referring to vehicle data.
VISSVehicle Information Service Specification — an API standard (W3C) for accessing VSS data over WebSocket or HTTP. KUKSA implements a gRPC variant.
COVESAConnected Vehicle Systems Alliance — industry consortium that maintains VSS, VISS, and other automotive data standards.
KUKSAEclipse KUKSA — the vehicle data broker project. The 'kuksa.val' databroker is the central in-vehicle VSS data store.
DatabrokerThe core KUKSA component that stores and serves vehicle signals. Other apps connect to it via gRPC to read or write values.
AnkaiosEclipse Ankaios — lightweight workload orchestrator for automotive ECUs. Think of it as a tiny Kubernetes for embedded systems.
ChariottEclipse Chariott — service discovery and programming model for vehicles. Allows apps to find and call services by name.
AgemoA publish/subscribe messaging system that is part of Eclipse Chariott.
LedaEclipse Leda — a complete SDV Linux distribution that bundles KUKSA, Ankaios, and other components into a ready-to-run image.
IbejiEclipse Ibeji — in-vehicle digital twin service. Manages a live model of vehicle state.
FreyjaPart of Eclipse Ibeji — synchronizes the in-vehicle digital twin with a cloud-side digital twin.
VelocitasEclipse Velocitas — SDK and toolchain for developing Vehicle Apps that interact with KUKSA.
Vehicle AppA small software component (microservice) running inside the vehicle that implements a specific feature using vehicle data.
ECUElectronic Control Unit — a small embedded computer inside the vehicle controlling a specific function (engine, brakes, seats, etc.).
CAN BusController Area Network — the dominant serial communication protocol used inside vehicles. All ECUs share a bus and broadcast messages by ID.
SOME/IPScalable service-Oriented Middleware over IP — AUTOSAR-defined middleware protocol for automotive Ethernet networks. Used for service calls and events.
DDSData Distribution Service — a publish/subscribe middleware standard (OMG) used in robotics and automotive applications for real-time data sharing.
MQTTMessage Queuing Telemetry Transport — lightweight publish/subscribe protocol commonly used for IoT and cloud connectivity.
gRPCGoogle Remote Procedure Call — a high-performance RPC framework using Protocol Buffers. KUKSA uses gRPC as its primary API.
ZenohA pub/sub protocol optimized for robotics and edge/cloud deployments. Used as a transport in some SDV components.
uProtocolEclipse uProtocol — a transport-agnostic messaging protocol for vehicles bridging MQTT, SOME/IP, Zenoh, and others.
ISO-TPISO 15765-2 Transport Protocol — a CAN-based transport layer for multi-frame messages. Used in vehicle diagnostics.
UDSUnified Diagnostic Services (ISO 14229) — the standard diagnostic protocol used by workshops to read fault codes and program ECUs.
Digital TwinA virtual representation of a physical vehicle or component that mirrors its real-time state.
AUTOSARAUTomotive Open System ARchitecture — industry-wide software architecture standard for ECUs. AUTOSAR Adaptive Platform targets high-performance domains.
OBD-IIOn-Board Diagnostics II — a standardized vehicle diagnostics interface present in all cars since the 1990s.
ADASAdvanced Driver Assistance Systems — systems like automatic emergency braking, lane keeping, adaptive cruise control.
SDVSoftware Defined Vehicle — a vehicle architecture where features are primarily defined and delivered through software rather than hardware.
Cyclomatic ComplexityA code metric measuring the number of independent paths through a function. Higher = more complex = harder to test and maintain. Aim for <10.
Nesting DepthHow many levels deep the most deeply nested code is (if inside if inside loop, etc.). Deep nesting hurts readability.
